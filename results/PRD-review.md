## 二、必须修订项（P0 — 影响开发实施）

### P0-1: 架构术语错误
- **当前位置**: 1.1、2.2、7.1、文档概要
- **问题**: 文档多处标注"CS 架构"，但实际描述的是 **B/S 架构**（Browser/Server，浏览器-服务器模式）。Spring Boot + Vue 3 的技术栈是典型的 Web 应用，不是传统的客户端-服务器架构。
- **修订建议**: 全文统一将"CS 架构"改为 **B/S 架构**。

### P0-2: 角色权限与用户关系缺失
- **当前位置**: 6.1、6.2 数据模型
- **问题**: 
  - 缺少学生与家长的绑定关系表（一个家庭可能有多个家长、多个孩子）
  - 数据模型中只有 `student_account`、`parent`、`task_record`、`product`，缺少 `user` 统一用户表和 `family_group` 家庭组表
  - RBAC 权限模型未落地到具体接口
- **修订建议**: 
  - 新增 `user` 表（统一认证）：id, username, password_hash, role(student/parent/admin), family_group_id
  - 新增 `family_group` 表：id, name
  - 明确家长与学生的 1:N 或 M:N 关系

### P0-3: 任务定义与任务记录分离
- **当前位置**: 4.2.1、6.2 task_record 表
- **问题**: 
  - PRD 既有预定义任务（课前预习、课后作业等），又提到"学生可自行设定任务"，但未定义两者的关系
  - `task_record` 中 `points` 字段由学生提交时填写，存在伪造积分风险
  - 缺少 `task_template`（任务模板）表，预定义任务的规则（积分值、完成标准、每日上限）无法在数据库中管理
  - 缺少 attachment 字段，凭证（图片或文字）无处存储
- **修订建议**:
  - 新增 `task_template` 表：id, name, type, base_points, extra_points_rule, daily_cap, category, description, standard
  - `task_record` 中的积分值应**由后端根据 task_template 规则自动计算**，不接受前端提交
  - 新增 `attachment_url` 字段存储凭证图片路径

### P0-4: 礼品兑换流程不完整
- **当前位置**: 4.2.3 礼品兑换管理
- **问题**: 
  - 兑换后积分如何扣减？立即扣除还是审核通过后扣除？
  - 兑换申请需要家长审核吗？还是学生直接扣积分兑换？
  - 缺少 `exchange_request`（兑换申请）表来追踪兑换流程
  - 冻结积分机制（frozen_points）只在数据模型中提到，业务逻辑中未定义使用场景
- **修订建议**:
  - 新增 `exchange_request` 表：id, student_id, product_id, points_cost, status(待确认/已发货/已完成/已取消), create_time, confirm_time
  - 兑换流程：学生提交申请 → 冻结对应积分 → 家长确认发货 → 实际扣除积分 → 状态更新
  - 明确"即时奖励"是否需要审核（建议即时奖励自动确认，中长期奖励需家长确认）

### P0-5: 图片/文件存储方案缺失
- **当前位置**: 4.2.1 任务打卡、非功能需求
- **问题**: 学生上传图片凭证，但未定义文件存储方案
- **修订建议**: 
  - 明确使用本地文件存储（上传到服务器 ./uploads 目录）或对象存储
  - 定义图片大小限制（建议单张 < 5MB）、格式限制（JPG/PNG）、存储路径规范
  - 后端需提供文件上传接口

## 三、强烈建议项（P1 — 影响产品质量）

### P1-1: 家务德育模块数据模型
- **当前位置**: 4.5 家务德育积分
- **问题**: 4.5 节定义了丰富的家务和品德积分规则，但 6.2 节数据模型中完全没有对应的表结构
- **修订建议**: 复用 `task_template` + `task_record`，通过 category 字段区分（学习/家务/品德/兴趣），或新增专用表

### P1-2: 统计可视化需求未明确
- **当前位置**: 3.2 家长端数据分析需求
- **问题**: 提到了"习惯养成统计"、"积分获取趋势"、"导出数据报表"，但没有定义具体统计维度、图表类型、导出格式
- **修订建议**:
  - 定义至少 3 种统计图表类型：日历打卡热力图、积分获取趋势折线图、任务完成率饼图
  - 数据报表导出格式：CSV 或 Excel
  - 时间维度：日/周/月

### P1-3: 淘宝链接导入技术可行性
- **当前位置**: 4.4 淘宝链接导入
- **问题**: 
  - 淘宝开放平台 API 需要企业资质、App Key、App Secret，个人用户难以申请
  - 短链接解析和淘口令解析需要额外的第三方服务
  - PRD 依赖淘宝 API，但没有 B 计划
- **修订建议**:
  - 提供手动录入商品信息作为降级方案（必填：商品名、积分要求、图片URL、链接）
  - 淘宝 API 作为可选增强功能，不作为 MVP 必需功能
  - 或考虑使用第三方爬虫/代理服务作为替代方案

### P1-4: 缺少消息通知机制
- **问题**: 学生完成任务提交后，家长如何及时知道？缺少通知机制
- **修订建议**: 系统内通知 + 可选的浏览器通知或邮件通知（MVP 阶段只需系统内通知红点提示）

### P1-5: 缺少数据初始化/种子数据
- **问题**: 系统首次使用时需要预置的任务模板、礼品数据等无来源
- **修订建议**: 提供 init_sql 脚本，预置 PRD 中定义的所有任务规则和默认礼品

## 四、优化建议（P2 — 锦上添花）

### P2-1: 积分连续打卡奖励
- **建议**: 增加连续打卡奖励机制（连续 7 天额外 +5 分，连续 30 天额外 +20 分），增强用户粘性

### P2-2: 任务完成提醒
- **建议**: 每日固定时间提醒学生打卡（如每晚 20:00 推送提醒）

### P2-3: 礼品心愿清单
- **建议**: 学生可设置"目标礼品"，系统自动计算还需多少积分，激励感更强

### P2-4: 家长操作日志
- **建议**: 所有家长端操作（尤其积分调整）增加审计日志表，防止纠纷

### P2-5: 批量审核确认对话框
- **建议**: 批量审核时需二次确认，显示本次发放积分总额，防止误操作

### P2-6: 数据模型补充字段
- **建议**:
  - `task_record` 增加 `parent_comment`（家长审核评语）
  - `product` 增加 `category` 字段（唱歌/手工/漫画/游乐场/其他）
  - `product` 增加 `is_active` 替代下架操作

## 五、技术架构评审意见

### 5.1 技术选型认可
- ✅ Spring Boot 3.0+ + SQLite 适合家庭场景，轻量部署合理
- ✅ Vue 3 + Vite 前端选型主流，开发效率高
- ✅ MyBatis-Plus 简化 CRUD，适合本项目规模
- ✅ Ehcache 本地缓存对单机部署友好

### 5.2 技术建议
- ⚠️ SQLite 在高并发写入时存在锁竞争，建议配置 WAL 模式（Write-Ahead Logging）
- ⚠️ 图片存储如果放在应用同目录下，需注意备份时包含 uploads 目录
- ✅ 数据库备份方案（SQLite Online Backup API）设计合理

## 九、补充的用户故事（User Stories）

### 学生端用户故事

#### US-S01：任务打卡
> **_as a** 四年级学生_，**_I want to_ 在系统中查看今日任务并提交打卡_，**_so that I can_ 获得积分奖励

**场景 1：提交文字说明打卡**
- 学生登录 → 查看"今日任务"列表 → 选择"课前预习" → 点击"打卡" → 输入文字描述（如"完成了语文和数学预习"） → 点击提交 → 显示"待审核"

**场景 2：上传图片凭证打卡**
- 学生登录 → 选择任务 → 点击"打卡" → 选择"上传照片" → 选取图片 → 点击提交 → 显示"待审核"

**场景 3：查看打卡状态**
- 学生在任务列表中看到颜色标记：灰色=未开始、黄色=待审核、绿色=已通过、红色=未通过

**场景 4：查看未通过原因**
- 学生点击查看被拒绝的打卡 → 显示家长填写的评语（如"凭证不完整，请补充数学预习照片"）

#### US-S02：积分查询
> **_as a** 四年级学生_，**_I want to_ 查看我的积分总数和获取明细_，**_so that I can_ 知道自己攒了多少分、怎么来的

**场景 1：查看积分总览**
- 学生登录后首页顶部显示当前可用积分（大数字 + 动画效果）
- 点击进入"积分明细"页面

**场景 2：查看积分获取历史**
- 列表按时间倒序展示：日期 + 任务名称 + 获得积分 + 状态
- 支持按"本周/本月/全部"筛选

**场景 3：查看冻结积分**
- 如果有兑换申请正在处理中，显示冻结积分金额及对应的礼品名称

#### US-S03：礼品兑换
> **_as a** 四年级学生_，**_I want to_ 用积分兑换心仪礼品_，**_so that I can_ 获得奖励

**场景 1：浏览并兑换即时奖励**
- 学生进入"兑换商城" → 查看"即时奖励"类礼品（≤15分） → 点击"兑换" → 确认 → 积分立即扣除（或冻结） → 显示"兑换成功" → 家长端自动确认

**场景 2：兑换中长期奖励**
- 学生浏览"K歌房体验"（60分） → 点击兑换 → 系统校验积分足够 → 冻结积分 → 提交申请 → 等待家长确认 → 显示"兑换申请已提交，等待家长确认"

**场景 3：积分不足时**
- 学生查看"全年漫画杂志订阅"（500分） → 积分不足 → 按钮显示"还差 XX 分" → 不可点击

#### US-S04：心愿单
> **_as a** 四年级学生_，**_I want to_ 收藏想要的礼物_，**_so that I can_ 随时查看还需要多少分

**场景：**
- 学生浏览礼品时点击"收藏"（⭐图标） → 进入"心愿单"查看 → 每个礼品显示"当前积分/需要积分"及进度条

### 家长端用户故事

#### US-P01：任务审核
> **_as a** 家长_，**_I want to_ 审核孩子提交的任务打卡_，**_so that I can_ 确认任务完成质量并发放积分

**场景 1：逐条审核**
- 家长登录 → 看到"待审核 3 项"红点提示 → 点击进入 → 查看每条打卡的凭证（文字/图片） → 点击"通过"或"拒绝" → 拒绝时填写评语 → 提交

**场景 2：批量审核**
- 家长勾选多个打卡记录 → 点击"批量通过" → 弹出确认框显示"将发放 XX 积分，确认？" → 确认后批量通过

#### US-P02：积分调整
> **_as a** 家长_，**_I want to_ 手动添加或扣除积分_，**_so that I can_ 灵活管理奖励和惩罚

**场景：扣分操作**
- 家长进入"积分管理" → 点击"调整积分" → 选择"扣分" → 输入分值 → 选择原因（作业未完成/说谎等） → 填写备注 → 确认 → 记录到积分流水

#### US-P03：商品管理
> **_as a** 家长_，**_I want to_ 上架和下架兑换商品_，**_so that I can_ 管理奖励池

**场景 1：手动创建商品**
- 家长进入"商品管理" → 点击"新增" → 填写名称、积分要求、描述 → 上传商品图片 → 选择分类 → 保存上架

**场景 2：淘宝链接导入**
- 家长粘贴淘宝商品链接 → 点击"解析" → 系统自动填充商品名称、图片、价格 → 家长修改积分要求后上架

#### US-P04：数据查看
> **_as a** 家长_，**_I want to_ 查看孩子的习惯养成统计_，**_so that I can_ 了解学习进展

**场景：**
- 家长进入"统计报表" → 查看本周打卡热力图 → 查看积分趋势折线图 → 选择时间范围 → 导出 CSV 报表

## 十、验收标准清单（Acceptance Criteria）

### 模块 AC-01：用户注册与登录
| 编号 | 标准描述 | 预期结果 |
|------|---------|---------|
| AC-01-01 | 使用正确的用户名和密码登录 | 登录成功，跳转到对应角色的首页 |
| AC-01-02 | 使用错误密码连续 5 次登录 | 账号被锁定，提示"账号已锁定，15分钟后再试" |
| AC-01-03 | 学生登录后看到学生端首页 | 显示积分、今日任务列表、底部导航 |
| AC-01-04 | 家长登录后看到家长端首页 | 显示待审核提示、快捷入口、数据概览 |
| AC-01-05 | 超过 30 分钟无操作 | 自动退出登录，跳转回登录页 |

### 模块 AC-02：任务打卡
| 编号 | 标准描述 | 预期结果 |
|------|---------|---------|
| AC-02-01 | 学生提交文字说明打卡 | 任务状态变为"待审核"，家长端出现待审核条目 |
| AC-02-02 | 学生上传图片凭证打卡 | 图片成功上传并在打卡记录中可预览 |
| AC-02-03 | 上传 > 5MB 的图片 | 系统拒绝上传并提示"图片大小不能超过 5MB" |
| AC-02-04 | 同一任务当天重复提交 | 系统提示"今日已提交该任务"，不允许重复提交 |
| AC-02-05 | 任务达到每日上限 | 继续提交时系统提示"今日该任务已达上限" |

### 模块 AC-03：任务审核
| 编号 | 标准描述 | 预期结果 |
|------|---------|---------|
| AC-03-01 | 家长通过审核 | 学生积分增加，学生端看到"已通过"，积分流水新增记录 |
| AC-03-02 | 家长拒绝审核并填写评语 | 学生端看到"未通过"及家长评语，积分不变 |
| AC-03-03 | 批量审核 5 条通过 | 5 条记录均变"已通过"，学生积分累加，流水记录 5 条 |
| AC-03-04 | 审核已审核过的任务 | 不可操作，按钮显示为"已通过"或"已拒绝" |

### 模块 AC-04：积分管理
| 编号 | 标准描述 | 预期结果 |
|------|---------|---------|
| AC-04-01 | 学生查看当前积分 | 显示准确的可用积分（总积分 - 冻结积分） |
| AC-04-02 | 学生查看积分流水 | 显示完整的积分变动记录，按时间倒序 |
| AC-04-03 | 家长手动加分 | 学生积分增加，流水中标注"家长手动添加"及原因 |
| AC-04-04 | 家长手动扣分 | 学生积分减少，流水中标注"家长扣除"及原因，积分不为负 |
| AC-04-05 | 积分不足时尝试兑换 | 系统提示"积分不足，还差 XX 分"，无法提交兑换 |

### 模块 AC-05：礼品兑换
| 编号 | 标准描述 | 预期结果 |
|------|---------|---------|
| AC-05-01 | 学生提交兑换申请（积分足够） | 对应积分被冻结，状态为"待确认" |
| AC-05-02 | 学生提交兑换申请（积分不足） | 系统阻止提交并提示积分不足 |
| AC-05-03 | 家长确认兑换 | 冻结积分转为实际扣除，兑换状态变更为"已发货" |
| AC-05-04 | 家长拒绝兑换 | 冻结积分释放回可用积分，状态变更为"已取消" |

## 十一、新增模块建议：新手引导

### US-G01：首次登录引导
> **_as a** 新用户_，**_I want to_ 了解系统的基本功能和操作流程_，**_so that I can_ 快速上手使用
