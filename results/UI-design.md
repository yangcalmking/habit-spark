## 一、设计规范文档 (Design System)

### 1.1 整体设计理念：两端差异化

本项目服务两端差异化明显，采用**独立视觉风格策略**：
- **学生端 (App/Web)**: “魔法成长空间” —— 温暖、圆润、游戏化、激励感。降低操作门槛，利用视觉反馈（动效）维持 11 岁女孩的兴趣。
- **家长端 (Manager Dashboard)**: “清晰高效管家” —— 专业、冷静、数据化。利用高密度布局、图表和批量操作提升管理效率。

### 1.2 学生端色彩与样式 (Cute Style)

**关键词：粉紫蓝、圆角、糖果色**

| 色彩定义 | 色值 (Hex) | 使用场景 |
| :--- | :--- | :--- |
| 品牌主色 | `#FF7EB3` (樱花粉) | 核心按钮、积分数字、主视觉元素 |
| 辅助色 1 | `#B166E6` (薰衣草紫) | 次级按钮、装饰背景、图标 |
| 辅助色 2 | `#7AD8FD` (天空蓝) | 进度条、高亮标签 |
| 状态 - 成功 | `#52D681` (薄荷绿) | 任务完成、审核通过 |
| 状态 - 警告 | `#FFC851` (芒果黄) | 待审核、连续中断风险 |
| 状态 - 失败 | `#FF6B6B` (草莓红) | 审核拒绝、错误 |
| 背景色 | `#FFF8F0` (奶油白) | 全局底色 (避免纯白刺眼) |

**样式规范：**
- **圆角**: 卡片 `24px`, 按钮 `50px` (胶囊型), 输入框 `16px`。杜绝尖锐直角。
- **字体**: 标题 `24px` / 正文 `18px` / 辅助 `14px`。最小字号 `16px` 确保易读。
- **阴影**: `0 8px 24px rgba(255, 120, 180, 0.2)` (柔和的粉色弥散阴影)。

### 1.3 家长端色彩与样式 (Clean Pro)

**关键词：蓝白灰、卡片、Element Plus 标准**

- **品牌色**: `#409EFF` (标准蓝)。
- **布局**: 左侧导航树，右侧内容区。卡片背景纯白 `#FFFFFF` 带细边框 `#EBEEF5`。
- **图表**: 使用 ECharts，色彩统一且克制（蓝 `#5470C6`、绿 `#91CC75`、橙 `#FAC858`）。

## 二、关键页面原型设计

### 2.1 学生端 (Student View)

#### 📄 P1. 首页 / 今日任务 (Home)
**核心目标**: 激励打卡，展示进度。

```text
┌──────────────────────────────────────┐
│ 🌸 嗨，小雨！今天是 3月20日 周五 🌸  │
├──────────────────────────────────────┤
│ 📊 我的积分                           │
│ ┌──────────────────────────────┐     │
│ │  ⭐ 128   🔥连签7天   ❄️冻结10 │   │
│ │ (主数字粉色大字体)             │    │
│ └──────────────────────────────┘     │
├──────────────────────────────────────┤
│ 📅 今日任务                           │
│ ┌──────────────────────────────┐     │
│ │ 📚 课前预习          [+4分]   │     │
│ │ 标准: 标生字,读英语 3 遍       │     │
│ │ 截止时间: 20:00              │     │
│ │       [ ✨ 去打卡 📝 ]      │     │  ← 胶囊主按钮
│ └──────────────────────────────┘     │
│ ┌──────────────────────────────┐     │
│ │ 🏠 整理房间          [+3分]   │     │
│ │ (学生自定义·审核中⏳)          │     │
│ │       等待妈妈审核...        │     │
│ └──────────────────────────────┘     │
├──────────────────────────────────────┤
│ 🛒 快捷入口                           │
│ ( 🎁 礼品兑换 ) ( 💖 心愿单 )        │
└──────────────────────────────────────┘
```
- **布局说明**: 顶部“个人资产卡片”展示总积分、连续天数、冻结积分（PM 评审 P0-4）。
- **任务卡片**: 高亮未完成的任务（粉色边框），进行中/已完成的任务灰度或显示已通过状态。
- **底部 Tab**: `🏠 任务` | `📊 积分` | `🛒 商城` | `👧 我的`。

#### 📄 P2. 任务打卡页 (Check-in Modal)
**核心目标**: 极简提交流程。

- **弹窗/抽屉式设计**: 避免页面跳转。
- **交互**:
    - 顶部显示任务名称及标准提醒（“完成标准：..."）。
    - **Tab 切换**：`[ 📝 文字记录 ]` | `[ 📸 图片上传 ]`。二选一即可。
    - 底部悬浮“提交打卡”按钮。
    - **反馈**: 提交成功触发“撒花”动效 🎉，提示“打卡成功！坐等妈妈审核”。

#### 📄 P3. 自定义任务设定页 (Custom Task)
**核心目标**: 赋予自主权，但流程严谨（Admin 新增需求）。

- **入口**: 首页底部 FAB (Floating Action Button) `+` 号。
- **字段设计**:
    - 任务名称：`[ 输入框 ]` (如：练习钢琴 20 分钟)
    - 所属类别：`( ) 学习` | `(x) 兴趣` | `( ) 其他`
    - 建议分值：`[ 数字滑块 1-20 分 ]` (注：提示“最终分值由妈妈决定哦”)
    - 完成频次：`[ 每天 ]`
- **底部操作**: `取消` | `✨ 提交给妈妈批准`。
- **提交后状态**: 在任务列表显示“待设定审核”，家长端对应出现“任务申请”。

#### 📄 P4. 积分查询页 (Points Detail)
**核心目标**: 账单清晰，理解变动。

- **顶部**: 积分余额大卡片。切换查看 `可用积分` | `冻结积分` | `总积分`。
- **列表**: 时间轴样式。
    - 绿色 `+10` (预习通过)
    - 灰色 `-50` (兑换魔法画笔)
    - 蓝色 `冻结 -60` (兑换 KTV 体验 - 待家长发货)
- **筛选**: 支持按“获得/消耗/冻结”筛选。

#### 📄 P5. 礼品兑换商城 (Shop)
**核心目标**: 逛得开心，兑换明确。

- **分类**: 顶部横向滚动 Tab `🎤 唱歌` | `🎨 手工` | `📚 漫画` | `🎢 游乐场`。
- **卡片瀑布流**:
    - 图片 + 名称 + 积分需求。
    - **按钮逻辑**:
        - 积分 ≥100%: `[ 🎁 立即兑换 ]`
        - 积分 <100%: `[ 🔒 还差 ?? 分 ]` (置灰)
        - 收藏态: 右上角显示 `♥` 实心。
- **交互**: 点击兑换弹出二次确认框“确定消耗 ?? 积分兑换 [玩具] 吗？”

#### 📄 P6. 心愿单页 (Wishlist)
**核心目标**: 目标可视化，激励感。

- **列表**: 展示收藏的礼品。
- **特色组件**: **进度条显示**。
    - `[图片] 乐高城堡 (800 分)`
    - `[========......] 78%`
    - “你已经走了 78%，加油！还需要 2 天！”

### 2.2 家长端 (Parent Dashboard)

#### 📄 P7. 首页 / 审核看板 (Dashboard)
**核心目标**: 一眼看清，一键处理。

```text
┌──────────────────────────────────┐
│ 📊 家长控制台                    │
│ 待办事项                         │
│ [ 🟢 3 ] 待审核任务 (需处理)     │
│ [ 🟡 1 ] 新任务申请 (需审批)     │
│ [ 🔴 0 ] 积分异常报警            │
├──────────────────────────────────┤
│ 📋 快速审核 (最近提交)           │
│ 1. 任务：复习英语 (20min)        │
│    凭证: [图片预览] 文字描述...   │
│    [ ✅ 通过 ]  [ ❌ ]          │
│ 2. 任务：整理房间 (自定义) ...    │
├──────────────────────────────────┤

│ [折线图区域]                      │
└──────────────────────────────────┘
```

#### 📄 P8. 任务审核列表 & 批量处理 (Audit)
**核心目标**: 高效，准确。

- **表格视图**:
    - 列：学生 | 任务名称 | 提交时间 | 凭证 (文字/图) | 状态 | 操作。
- **操作**:
    - 支持多选复选框。
    - **批量通过**: 点击后弹出 `Dialog`：“确认通过选中的 5 个任务吗？共发放 +25 积分。”
    - **拒绝**: 必须输入拒绝理由（必填校验），如“字迹潦草”，学生端同步可见。

#### 📄 P9. 商品管理 (Product Mgmt)
**核心目标**: 快速上架，链接解析。

- **表格**: 商品图、名称、所需积分、库存、状态 (上架/下架)、操作 (编辑/删除)。
- **新增/编辑弹窗**:
    - `淘宝链接解析`: `[ 粘贴 URL 在此 ] 🔍 解析`。
    - 解析后自动带入：标题、图片、价格（仅供参考）。
    - 手动修正区：`设置所需积分` `库存数`。

#### 📄 P10. 数据统计分析 (Stats)
**核心目标**: 趋势洞察。

1. **打卡热力图**: GitHub 风格网格，颜色越深打卡越多。查看坚持度。
2. **积分构成饼图**: “学习类占比 70%"vs“兴趣类占比 30%"。
3. **能力雷达图**: 预习/复习/作业/兴趣 维度的得分分布。
4. **导出**: `📤 导出本周报表 (CSV)`。

## 三、核心交互与状态流转说明

### 3.1 积分状态流转 (关键 P0-4)

| 动作 | 学生端表现 | 积分账户变化 | 家长端表现 |
| :--- | :--- | :--- | :--- |
| 兑换申请提交 | 按钮变“处理中”，出现倒计时 | **冻结**: -N | **新增**：待确认兑换单 |
| 家长确认发货 | 状态变“已发货/待收货”，提示“妈妈发货啦” | **扣除**: 冻结转扣除，Available 不变已扣过 | 记录消耗流水，库存-N |
| 家长拒绝/取消 | 礼物回到购物车，进度条回退 | **解冻**: 冻结 +N 回滚 | 记录取消原因 |

### 3.2 任务自定义申请流转

1. **学生**: 提交自定义任务申请（名称、分数建议）。
2. **家长**: 审核看板出现“任务申请”。
3. **操作**: 家长可“批准”（可直接修改分数，如学生申请 5 分，家长认为工作量小改为 3 分并批准），或“驳回”。
4. **学生**: 收到系统通知“你添加的【洗碗】已被批准/驳回”。

### 3.3 动效与反馈规范

- **即时反馈**: 所有点击操作（提交、点赞、兑换）必须在 200ms 内给出微动效。
- **加载态**: 避免使用传统的“转圈圈”，使用“可爱的云朵飘动”或“小星星闪烁”。
    - *家长端*: 使用标准的 Skeleton Screen (骨架屏)。

## 四、两端风格对比总结

| 对比维度 | 学生端 🌸 | 家长端 💼 |
| :--- | :--- | :--- |
| **设计语言** | **Material Rounded / Cute 3D** | **Element Plus / Ant Design Pro** |
| **色彩倾向** | 暖色调 (粉/紫/橙) | 冷/中性色调 (蓝/白/灰) |
| **图形风格** | 圆角大 (`24px+`)，装饰多 | 直角/小圆角 (`4px`)，极简 |
| **信息密度** | 低 (大卡片，大字，留白多) | 高 (表格密集，数据紧凑) |
| **核心交互** | 滑动、点击大按钮、沉浸式弹窗 | 点击链接、表格排序、批量勾选 |

## 五、响应式设计建议

**适用设备**: 学生端侧重于 平板 (10 寸) 及 笔记本; 家长端侧重于 桌面显示器 (PC)。

**Student Responsive Breakpoints**:
- **Mobile (< 600px)**:
    - 卡片从 Grid 双列变为 单列垂直布局。
    - 底部导航栏保留。
- **Desktop (> 1200px)**:
    - 居中显示主内容区 (Max-width 800px)，两侧留白或使用装饰侧边栏 (如显示角色形象)。

**Parent Responsive Breakpoints**:
- **Standard**: 固定左侧菜单栏，不折叠。
- **Small Screen (Tablet)**:
    - 左侧菜单自动收起为图标模式 (Icon Only)。
    - 表格增加横向滚动条，不压缩列宽导致内容换行混乱。
